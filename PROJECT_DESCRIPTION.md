# Goszakup Analytics — Аналитика государственных закупок Казахстана

## Обзор

Инструмент для аналитики государственных закупок Казахстана, работающий с официальным API портала **goszakup.gov.kz** (Единая система электронных государственных закупок РК).

Проект подключается к GraphQL API портала госзакупок и выгружает данные по конкретной организации-заказчику (по её БИН), после чего формирует Excel-отчёты.

---

## Функциональность

### 1. Выгрузка договоров (`get_contracts.py`)

Получает **полный список договоров** организации за указанный финансовый год. Для каждого договора собирает:

- Номер, дата подписания, описание
- Суммы: плановая, без НДС, фактическая
- Поставщик, статус, тип договора, способ закупки
- Номер закупки (объявления)

**Результат** — Excel-файл со всеми договорами в виде таблицы.

### 2. Аналитический отчёт (`generate_report.py`)

Самый объёмный скрипт. Формирует **сводный аналитический отчёт** в Excel, который включает:

- **Общую сводку**: фактическая сумма закупок, экономия (план минус факт)
- **Разбивку по видам предмета**: товары, работы, услуги — с суммами
- **Таблицу 1**: закупки по способам (конкурс, из одного источника, аукцион и т.д.) — плановые суммы, фактические суммы, экономия (с НДС и без)
- **Таблицу 2**: перекрёстная таблица «способ закупки × вид предмета» — количество договоров и суммы
- **Количество расторгнутых договоров** (отдельный запрос по статусам 340, 350)
- **Сводку по объявлениям** за период
- Поддержка фильтрации **по кварталам**

### 3. Сводка по объявлениям (`get_announcements.py`)

Получает **объявления о закупках** за указанный период и группирует их по способу закупки (конкурс, аукцион, из одного источника и т.д.). Выводит таблицу с количеством объявлений по каждому способу.

## Архитектура

```
main.py                    — CLI-меню для выбора действия
src/config.py              — Конфигурация: токен, БИН, год, статусы, пути
src/get_contracts.py       — Выгрузка сырых данных по договорам
src/generate_report.py     — Аналитический отчёт с агрегацией
src/get_announcements.py   — Статистика по объявлениям
```

### Технические детали

| Параметр | Значение |
|---|---|
| **API** | GraphQL v3 на `https://ows.goszakup.gov.kz/v3/graphql` |
| **Авторизация** | Bearer-токен |
| **Пагинация** | Курсорная (по `lastId`), до 200 записей за запрос |
| **Конфигурация** | `.env` файл (токен, БИН, год, квартал, даты) |
| **Выходной формат** | Excel (.xlsx) с форматированием (шрифты, рамки, цветные заголовки, настройки печати) |

### Зависимости

- `requests` — HTTP-запросы к API
- `pandas` — обработка табличных данных
- `openpyxl` — создание и форматирование Excel-файлов
- `python-dotenv` — загрузка переменных окружения из `.env`

## Конфигурация

Основные параметры задаются в файле `.env`:

| Переменная | Описание | Пример |
|---|---|---|
| `TOKEN` | API-токен с портала goszakup.gov.kz | `eyJhbGciOi...` |
| `BIN_COMPANY` | БИН организации-заказчика | `020240003361` |
| `FIN_YEAR` | Финансовый год | `2024` |
| `QUARTER` | Квартал (1–4, пусто = весь год) | `1` |
| `DATE_FROM` | Начало периода для объявлений | `2024-01-01` |
| `DATE_TO` | Конец периода для объявлений | `2024-12-31` |

### Статусы договоров

В аналитический отчёт включаются договоры со статусами:

- **390** — Исполнен
- **375** — Частично исполнен
- **190** — Действует

Расторгнутые договоры считаются отдельно:

- **340** — Расторгнут в одностороннем порядке
- **350** — Расторгнут по соглашению сторон

---

## Особенности расчёта

- **Экономия** = Плановая сумма − Фактическая сумма. Если фактическая сумма не указана или равна 0, используется сумма договора.
- **НДС** рассчитывается по ставке 12% для всех договоров (независимо от статуса плательщика НДС поставщика).
- **Фактическая сумма без НДС**: для плательщиков НДС указана сумма без НДС, для неплательщиков — без изменений.

---

## Целевая аудитория

Типичный пользователь — сотрудник отдела закупок государственной организации, которому нужно готовить **квартальные/годовые отчёты** об итогах проведённых закупок:

- Сколько потратили
- Какая экономия
- Сколько договоров по каждому способу закупки
- Сколько расторгнуто
